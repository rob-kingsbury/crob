# Crob Development Roadmap

**Version:** 0.2 (Python Rewrite with Web Interface)
**Date:** 2025-01-08

---

## Current State (v0.1)

**What exists:**
- PHP implementation of core concepts
- `.crob` text format for human-readable facts
- Basic web scraping via DuckDuckGo
- String-based query (exact match, case-insensitive, substring)
- Curiosity queue with priority/depth tracking
- CLI interface (`crob.php`)

**What's missing:**
- Autonomous learning (only learns when asked)
- Confidence dynamics (facts stored but not reinforced/degraded)
- Contradiction detection (can hold conflicting beliefs)
- Semantic retrieval (only string matching)
- Web interface (CLI only)
- Working memory limits (unlimited retrieval)

## Risk Mitigation

### Technical Risks

**Risk:** HDV adds complexity without benefit
- **Mitigation:** Phase 2 includes decision point to remove if not useful

**Risk:** Autonomous learning produces garbage
- **Mitigation:** Phase 0 validation before committing to web interface

**Risk:** Web scraping rate limits / blocks
- **Mitigation:** Polite delays, respect robots.txt, fallback to manual teaching

**Risk:** Knowledge graph becomes too large (performance)
- **Mitigation:** Pagination, lazy loading, confidence-based pruning

### Strategic Risks

**Risk:** No one uses it (not actually useful)
- **Mitigation:** Build for personal use first, share only if valuable

**Risk:** Scope creep (too many features)
- **Mitigation:** Strict phase gates, MVP focus

**Risk:** Maintenance burden
- **Mitigation:** Good documentation, modular architecture

## Open Questions to Resolve

### Design Decisions
1. **HDV dimensions:** 10K bits? 5K? Trade-off: collision vs performance
2. **Confidence decay:** Should unused facts degrade? If so, how?
3. **Working memory size:** 7 items (Miller's law) or dynamic?
4. **Research frequency:** Every 60s? Adaptive based on queue size?

### Implementation Choices
1. **Database:** Stick with files or add SQLite/PostgreSQL for queries?
2. **NLP library:** spaCy (heavy) vs simpler regex patterns?
3. **Frontend framework:** Alpine.js (minimal) vs React (full-featured)?
4. **Vector storage:** In-memory or persistent binary file?

### Product Direction
1. **Target users:** Personal tool? Open-source project? Research artifact?
2. **Domain focus:** General knowledge? Specific field?
3. **Business model:** Free/open-source? Premium features? None?

## Final Note

**This is a pet project with potential.**

The goal isn't to compete with ChatGPT or build a startup.

The goal is to explore:
- What does cognitively faithful AI look like?
- Can transparent knowledge systems be useful?
- Does autonomous curiosity produce valuable learning?

If the answers are "interesting," "yes," and "sometimes," then Crob succeeds.

If the answers are "meh," "no," and "no," then we learned what doesn't work—also valuable.

**Build for curiosity, not for scale.**
